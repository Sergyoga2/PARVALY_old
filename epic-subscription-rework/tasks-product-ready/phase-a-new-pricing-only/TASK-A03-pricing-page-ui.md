## A) Краткое описание

Заменить текущую страницу тарифов на новую с 4 тарифами: 1 мес / 6 мес / 12 мес / 3 года. Флагманский продукт — **3-летняя подписка**: она должна быть представлена как премиальное, эксклюзивное предложение с максимальной визуальной приоритетностью. Тариф 3 года показал наивысшие продажи — задача страницы максимально стимулировать его покупку.

Страница должна ясно показывать ценность каждого тарифа, визуально вести пользователя к выбору 3-летнего тарифа, показывать экономию. В будущем (Phase B) на странице появится CTA для trial — под него необходимо подготовить место.

Страница предназначена для всех пользователей: неавторизованных (привлечение), авторизованных без подписки (конверсия), с подпиской (информация и upgrade), с legacy-подпиской (информация о новых тарифах).

---

## B) Scope / Out of Scope

### Scope
- Вёрстка новой страницы с 4 карточками тарифов (1 мес / 6 мес / 12 мес / 3 года)
- Для каждого тарифа: цена/мес, общая стоимость, скидка в %, экономия в ₽
- **Премиум-оформление тарифа 3 года** — увеличенная карточка, акцентный цвет, бейдж «Лучшее предложение», иконка/лейбл «Флагман», анимация или тень для привлечения внимания
- Маркировка 6 мес — бейдж «Выбор большинства»
- Маркировка 12 мес — бейдж «Максимум выгоды»
- Блок социального доказательства на карточке 3 года: «Самый популярный среди наших пользователей», счётчик или % выбравших
- Feature flag для переключения между старой и новой страницей (старая не удаляется)
- Адаптивность: desktop, tablet, mobile
- Подготовка скрытого блока для Trial CTA (активируется в Phase B)
- Различное отображение для разных состояний пользователя (см. сценарии)

### Out of Scope
- Trial CTA (активация — Phase B)
- Payment flow / checkout (A04)
- Cancellation flow (Phase C)

## C) Бизнес-правила и состояния

### Данные карточек

| Тариф | Бейдж | Визуальный стиль | Цена/мес | Скидка | Итого | Экономия |
|-------|-------|-------------------|----------|--------|-------|----------|
| 1 мес | — | Обычная карточка | 3 900 ₽ | — | — | — |
| 6 мес | Выбор большинства | Выделенная карточка | 2 900 ₽ | -26% | 17 400 ₽ | 6 000 ₽ |
| 12 мес | Максимум выгоды | Выделенная карточка | 2 400 ₽ | -38% | 28 800 ₽ | 18 000 ₽ |
| **3 года** | **Лучшее предложение** | **Премиум-карточка: увеличенный размер, акцентный цвет, тень/glow, иконка эксклюзивности** | **1 900 ₽** | **-51%** | **68 400 ₽** | **72 000 ₽** |

### Визуальная иерархия карточки «3 года»

Карточка 3-летнего тарифа должна визуально доминировать на странице:
- **Размер**: на 15-20% крупнее остальных карточек
- **Цвет**: акцентный/премиальный (золотой или градиент), отличается от остальных карточек
- **Бейдж**: «Лучшее предложение» — крупный, контрастный
- **Социальное доказательство**: блок «Самый популярный среди наших пользователей» или «Выбирают 67% пользователей»
- **CTA-кнопка**: крупнее, другой цвет/стиль, текст «Оформить со скидкой 51%»
- **Дополнительный акцент**: тонкая анимация (пульсация тени, shimmer) или визуальный эффект, привлекающий внимание
- **Экономия**: крупно вынесена «Вы экономите 72 000 ₽»

### Порядок отображения

- Desktop: 1 мес → 6 мес → 12 мес → **3 года** (крайняя правая, увеличенная)
- Mobile: **3 года первая** (сверху, как hero-карточка) → 12 мес → 6 мес → 1 мес

### Состояния пользователя → отображение

| Состояние пользователя | Что видит |
|------------------------|-----------|
| Неавторизованный | 4 тарифа, CTA → redirect на регистрацию |
| Авторизованный, нет подписки | 4 тарифа, CTA → checkout. Карточка 3 года с доп. акцентом «Рекомендуем» |
| Active подписка (new) | Текущий тариф выделен, остальные «Перейти». На 3-летнем — «Перейти на максимум» |
| Active подписка (legacy 3-year) | Баннер «У вас тариф с максимальной выгодой» + информация о продлении |
| Active подписка (другой legacy) | Баннер «У вас архивный тариф» + 4 новых тарифа, акцент на 3-летний |
| Cancelled (доступ есть) | 4 тарифа, «Переоформить». На 3-летнем — «Вернуться с максимальной выгодой» |
| Expired | 4 тарифа, «Оформить». На 3-летнем — «Оформить со скидкой 51%» |

### Адаптивность
- Desktop (>1024px): 4 карточки в ряд, карточка 3 года визуально крупнее
- Tablet (768-1024px): 2×2, карточка 3 года занимает верхнюю правую позицию
- Mobile (<768px): вертикальный список, **карточка 3 года первая** (hero-позиция)

## D) Пользовательские и системные сценарии

1. **Given** новый неавторизованный пользователь, **When** открывает pricing page, **Then** видит 4 карточки тарифов с корректными ценами, скидками и экономией, карточка 3 года визуально доминирует.

2. **Given** feature flag выключен, **When** пользователь открывает pricing page, **Then** видит старую страницу тарифов.

3. **Given** feature flag включен, **When** пользователь открывает pricing page, **Then** видит новую страницу с 4 тарифами, 3-летний тариф выделен как премиум.

4. **Given** тариф 3 года, **When** отображается на странице, **Then** имеет бейдж «Лучшее предложение», увеличенный размер, акцентный цвет, блок социального доказательства и крупную CTA-кнопку.

5. **Given** тариф 6 мес, **When** отображается на странице, **Then** имеет бейдж «Выбор большинства».

6. **Given** мобильное устройство, **When** открывает pricing page, **Then** карточки отображаются вертикально, **карточка 3 года первая** (hero-позиция).

7. **Given** legacy-подписчик (не 3-year), **When** открывает pricing page, **Then** видит баннер о текущем тарифе (с пометкой «архивный») и все 4 новых тарифа с акцентом на 3-летний.

8. **Given** legacy-подписчик с 3-year тарифом, **When** открывает pricing page, **Then** видит позитивный баннер «У вас тариф с максимальной выгодой» и информацию о продлении.

9. **Given** пользователь с active подпиской на новый тариф, **When** открывает pricing page, **Then** текущий тариф выделен, на 3-летнем кнопка «Перейти на максимум».

10. **Given** неавторизованный пользователь, **When** нажимает CTA на тарифе, **Then** redirect на регистрацию с возвратом на pricing page.

11. **Given** API тарифов не отвечает, **When** страница загружается, **Then** отображаются cached/fallback данные.

12. **Given** авторизованный пользователь без подписки, **When** открывает pricing page, **Then** на карточке 3 года видит дополнительный лейбл «Рекомендуем».

## E) Acceptance Criteria

- [ ] 4 карточки тарифов (1 мес / 6 мес / 12 мес / 3 года) отображаются с корректными ценами, скидками и экономией
- [ ] Карточка 3 года оформлена как премиум: увеличенный размер, акцентный цвет, бейдж «Лучшее предложение», блок социального доказательства
- [ ] CTA-кнопка на карточке 3 года крупнее и акцентнее остальных, текст отражает скидку
- [ ] Карточка 6 мес имеет бейдж «Выбор большинства»
- [ ] Карточка 12 мес имеет бейдж «Максимум выгоды»
- [ ] На мобильных карточка 3 года отображается первой (hero-позиция)
- [ ] Feature flag: при выключении показывается старая страница, при включении — новая
- [ ] Страница корректно отображается на desktop, tablet и mobile
- [ ] Для каждого состояния пользователя отображение соответствует таблице состояний
- [ ] Legacy-подписчик с 3-year тарифом видит позитивный баннер о своём тарифе
- [ ] Другие legacy-подписчики видят баннер «архивный тариф» + акцент на 3-летний новый
- [ ] Скрытый блок для Trial CTA присутствует (для Phase B)
- [ ] CTA для неавторизованных ведёт на регистрацию
- [ ] Старая страница тарифов не удалена (для rollback)

## G) Риски и допущения (Assumptions)

### Допущения
- Дизайн-макеты будут предоставлены до начала реализации (с приоритетом на дизайн карточки 3 года)
- Данные по тарифам берутся из API (не захардкожены)
- Старая страница остаётся для быстрого rollback через feature flag
- 3-летний тариф — ключевой продукт; дизайн и UX-копирайт для него согласуются отдельно

### Риски
- Задержка дизайн-макетов → начинать с wireframe, итерировать
- Feature flag инфраструктура может быть не готова → уточнить у команды
- Премиум-оформление 3-летнего тарифа может выглядеть навязчиво → A/B тест разных уровней акцента
