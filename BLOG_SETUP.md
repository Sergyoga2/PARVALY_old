# Настройка блога PARVALY

## Структура блога

Блог настроен для двух языков с автоматическим управлением статьями через JSON файл.

### Структура папок:
```
/blog/
  ├── index.html                          # Главная страница блога (EN)
  └── improve-local-seo-2026.html        # Примерная статья (EN)

/ru/blog/
  ├── index.html                          # Главная страница блога (RU)
  └── uluchshit-lokalnoe-seo-2026.html   # Примерная статья (RU)

/assets/
  ├── css/blog.css                       # Стили блога
  └── js/blog.js                         # JavaScript для динамического списка

/blog-data.json                          # База данных статей
```

## Как добавить новую статью

### Шаг 1: Добавить метаданные в blog-data.json

Откройте `blog-data.json` и добавьте информацию о новой статье:

```json
{
  "articles": {
    "en": [
      {
        "id": "your-article-slug",
        "title": "Your Article Title",
        "slug": "your-article-slug",
        "description": "Short description for preview",
        "author": "PARVALY Team",
        "date": "2026-02-02",
        "category": "SEO",
        "tags": ["SEO", "Local Business", "Marketing"],
        "image": "/assets/blog/your-image.jpg",
        "published": true
      }
    ],
    "ru": [
      {
        "id": "vash-article-slug",
        "title": "Заголовок вашей статьи",
        "slug": "vash-article-slug",
        "description": "Краткое описание для превью",
        "author": "Команда PARVALY",
        "date": "2026-02-02",
        "category": "SEO",
        "tags": ["SEO", "Локальный бизнес", "Маркетинг"],
        "image": "/assets/blog/your-image.jpg",
        "published": true
      }
    ]
  }
}
```

**Важно:**
- `id` и `slug` должны быть уникальными
- `slug` используется в URL статьи
- `published: false` скроет статью из списка
- `date` в формате YYYY-MM-DD
- `image` опционален

### Шаг 2: Создать HTML файл статьи

1. **Для английской версии:**
   - Скопируйте файл `/blog/improve-local-seo-2026.html`
   - Переименуйте в `/blog/your-article-slug.html`
   - Измените заголовок, контент и метаданные

2. **Для русской версии:**
   - Скопируйте файл `/ru/blog/uluchshit-lokalnoe-seo-2026.html`
   - Переименуйте в `/ru/blog/vash-article-slug.html`
   - Измените заголовок, контент и метаданные

### Шаг 3: Отредактировать контент статьи

В HTML файле статьи измените:

1. **Meta теги** (в `<head>`):
   ```html
   <meta name="description" content="Ваше описание">
   <title>Ваш заголовок — Блог PARVALY</title>
   ```

2. **Заголовок статьи** (в `<article>`):
   ```html
   <h1>Ваш заголовок статьи</h1>
   ```

3. **Контент** (в `<div class="article-content">`):
   - Используйте `<h2>` для основных разделов
   - Используйте `<h3>` для подразделов
   - Используйте `<p>`, `<ul>`, `<ol>` для текста и списков
   - Добавьте изображения с `<img>`

4. **Теги** (в `<div class="article-tags">`):
   ```html
   <a href="/blog/?tag=seo" class="tag">SEO</a>
   ```

### Шаг 4: Проверить результат

1. Откройте `/blog/` или `/ru/blog/` в браузере
2. Новая статья должна автоматически появиться в списке
3. Проверьте фильтры по категориям
4. Проверьте саму статью

## Категории и фильтры

Доступные категории (можно расширять):
- SEO
- Google Ads
- Social Media
- Marketing
- Google Maps

Фильтры на главной странице блога работают автоматически на основе `category` и `tags` в JSON.

## SEO оптимизация

Каждая статья должна включать:

1. **Meta теги:**
   - `<meta name="description">` - до 160 символов
   - `<meta name="keywords">` - релевантные ключевые слова
   - Open Graph теги для социальных сетей

2. **Структурированный контент:**
   - H1 - один на страницу (заголовок статьи)
   - H2 - для основных разделов
   - H3 - для подразделов

3. **URL структура:**
   - Чистые URL: `/blog/article-slug.html`
   - Используйте дефисы, а не подчеркивания
   - Держите slug коротким и описательным

4. **Внутренние ссылки:**
   - Ссылайтесь на другие статьи блога
   - Ссылайтесь на страницы услуг
   - Используйте CTA в конце статей

## Добавление ссылок на блог в навигацию

Ссылки на блог уже добавлены в:
- `/blog/index.html`
- `/ru/blog/index.html`
- Примерные статьи блога

**Чтобы добавить ссылку "Blog" в другие страницы сайта:**

1. **Для английских страниц**, добавьте в навигацию:
   ```html
   <a href="/blog/">Blog</a>
   ```

2. **Для русских страниц**, добавьте:
   ```html
   <a href="/ru/blog/">Блог</a>
   ```

Места для добавления:
- В `<nav class="nav">` (десктоп меню)
- В `<nav class="mobile-nav">` (мобильное меню)
- В футере в разделе "Quick Links" / "Быстрые ссылки"

## Изображения для блога

Рекомендации:
- Размер: 1200x630 пикселей (для Open Graph)
- Формат: JPG или PNG
- Оптимизируйте размер файла (< 200KB)
- Храните в `/assets/blog/`
- Используйте описательные имена файлов

## Дополнительные возможности

### Newsletter форма
В каждой статье есть форма подписки. Для её работы нужно:
1. Настроить обработчик формы на сервере
2. Или интегрировать с сервисом email рассылок (Mailchimp, ConvertKit и т.д.)

### Связанные статьи
Автоматически показываются статьи из той же категории. Логика в `/assets/js/blog.js`.

### Комментарии
Для добавления комментариев можно интегрировать:
- Disqus
- Facebook Comments
- Собственная система комментариев

## Советы по контенту

1. **Длина статьи:** 1500-2500 слов оптимально для SEO
2. **Структура:**
   - Введение (почему это важно)
   - Основные пункты (2-6 разделов)
   - Заключение
   - CTA (призыв к действию)
3. **Ключевые слова:** используйте естественно, не переспамьте
4. **Изображения:** минимум одно изображение на статью
5. **Внутренние ссылки:** 2-5 ссылок на другие статьи или страницы
6. **CTA:** всегда завершайте статью призывом к действию

## Публикация статьи

1. Добавьте запись в `blog-data.json` с `"published": false`
2. Создайте и отредактируйте HTML файл статьи
3. Проверьте статью локально
4. Когда готово, измените на `"published": true`
5. Закоммитьте и запушьте изменения
